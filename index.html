vue-computed 计算属性

发表于 2019-06-20 | 分类于 vue学习 |

计算属性 解决的问题

  • 避免 模版上出现复杂的业务逻辑

何谓计算属性

  • 通过属性计算得来的属性，这个对象里存放的都是方法

    注意事项：

    • computed是一个属性，而不是方法。[在写法上应注意 computed:{ code…对象 }]
    • computed里面的方法在调用的时候不需要加括号(); [加了会提示 xxx不是一个function ]
    • 只要data中的属性发生了变化，那么就会触发computed里面相对应的方法;
    • computed具有缓存功能,若属性不改变则方法不会触发，而是从缓存中读取最终的结果;
    • computed里面的方法必须return出一个返回值

计算属性 computed 与方法 methods 之间的区别

  • computed执行不需要加括号，methods需要。在选择上，考虑类似需要通过点击事件执行的 使用methods方法，考虑数据转换后直接渲染使用计算属性（各种功能实现，最后显示），避免在模版上写过多的逻辑
  • computed具有缓存功能,计算属性是基于它们的依赖进行缓存的。计算属性只有在它的相关依赖发生改变时才会重新求值.

谈谈你对react生命周期中shouldComponentUpdate的理解

react性能优化非常重要的一环。组件接受新的state或者props时调用，我们可以设置在此对比前后两
个props和state是否相同，如果相同则返回false阻止更新，因为相同的属性状态一定会生成相同的
dom树，这样就不需要创造新的dom树和旧的dom树进行diff算法对比，节省大量性能，尤其是在
dom结构复杂的时候

谈谈你对虚拟DOM的理解，以及好处？

虚拟DOM：虚拟DOM其实就是js对象
虚拟DOM提高了react的性能，每次数据更新后都会重新计算上虚拟DOM，并和上一次的虚拟DOM
进行对比，对方法变化的部分进行批量更新。react中也提供了shouldComponentUpdate生命周期
的回调，来减少数据变化后不必要的虚拟DOM对比过程。保证性能。

谈谈你对flux的理解

利用单项数据流的方式来组合react的视图组件，它是一种模式，而不是一种框架简单来说flux可以帮我们解决非父子组件之间传值

组件传值的几种方式

父传子：父组件通过props传递给子组件。
子传父：父组件通过props向子组件传入一个方法，子组件通过调用该方法，将数据以参数形式传递给父组件
兄弟组件传值：

  • 子组件先传给父组件，父组件再传给那个子组件
    * 状态提升
    • 观察者模式

要点
  • 应用中所有的 state 都以一个对象树的形式储存在一个单一的 store 中。
  • 惟一改变 state 的办法是触发 action，一个描述发生什么的对象。

  • 为了描述 action 如何改变 state 树，你需要编写 reducers。

    • 把要做的修改变成一个普通对象，这个对象被叫做 action，而不是直接修改 state。
    • 然后编写专门的函数 reducer , 来决定每个 action 如何改变应用的 state，这个函数被叫做 reducer。

动机

  • 随着 JavaScript 单页应用开发日趋复杂，JavaScript 需要管理比任何时候都要多的 state （状态）。 这些 state 可能包括服务器响应、缓存数据、本地生成尚未持久化到服务器的数据，也包括 UI 状态，如激活的路由，被选中的标签，是否显示加载动效或者分页器等等。
  • 我们总是将两个难以厘清的概念混淆在一起：变化和异步。 我称它们为曼妥思和可乐。如果把二者分开，能做的很好，但混到一起，就变得一团糟。

三大原则

  • 唯一数据源（Single Source of Truth）
  • 保持状态只读（State is read-only）
  • 数据改变只能通过纯函数完成（Changes are made with pure functions）
单一数据源

整个应用的 state 被储存在一棵 object tree 中，并且这个 object tree 只存在于唯一一个 store 中。

State 是只读的

惟一改变 state 的方法就是触发 action，action 是一个用于描述已发生事件的 普通 对象 。
