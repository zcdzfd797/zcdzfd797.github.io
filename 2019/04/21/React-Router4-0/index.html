Switch

因为4.0的路由允许我们匹配多个路由，例如当访问 /admin/ui时，会匹配如图符合条件的所有路由，实际上我们只需要匹配一个，此时在外嵌套Switch

Redirect 重定向 exact 严格匹配

采用 <Switch>，只有一个路由会被渲染，并且总是渲染第一个匹配到的组件。因此，在第一个路由中，还是需要使用 exact，否则，当我们渲染 ‘/users’ 或 ‘/users/add’ 时，只会显示匹配 ‘/‘ 的组件

Switch与exact的区别？
  • switch 匹配首个符合条件的路由，exact用于精确匹配

this.props.children

  • this.props 对象的属性与组件的属性一一对应，但是有一个例外，就是 this.props.children 属性。它表示组件的所有子节点
  • 〇 ♡〇 this.props.children 的值有三种可能：如果当前组件没有子节点，它就是 undefined ;如果有一个子节点，数据类型是 object ；如果有多个子节点，数据类型就是 array 。所以，处理 this.props.children 的时候要小心。 React 提供一个工具方法 React.Children 来处理 this.props.children 。我们可以用 React.Children.map 来遍历子节点，而不用担心 this.props.children 的数据类型是 undefined 还是 object。
  • 嵌套路由

    访问 / 时 返回的是Main组件 因此在Main中再添加一层路由 this.props.children ,在router文件原来的{Main}处做修改为 render={ ()=>{ <需要渲染的组件></需要渲染的组件> }} ,此时可以在 需要渲染的组件内再嵌套一个路由。

使用嵌套路由时，最好用render返回一个主页面，在主页面中再实现嵌套路由的跳转

  • 注意事项：

    父节点勿用exact精准匹配，会导致子路由失效
