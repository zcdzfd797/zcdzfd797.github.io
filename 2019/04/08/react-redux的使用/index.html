store相当于图书馆管理员，他并不知道数据该如何存储，他需要一个本子帮他管理这些内容，所以我们会在store中创建一个reducer

引入ruducer，创建createStore时，将reducer传给这个方法

  • src/store/reducer.js
  • reducer 是一个纯函数，它接收两个参数 state 和 action ，默认情况下把 state 状态返回出去
  • 数据有时会有初始值，我们会定义一个 defaultState ,当 State 触发时让它默认等于 defaultState 。

问题：如何想在 TodoList.js 中使用store中的数据（react-redux方法）

  • src/index.js
核心api — Provider

Provider : 此提供器，如果连接store，那么Provider内所有组件都有能力获取到store里的内容。

  • src/TodoList.js
核心api — connect
  • connect : 让todolist组件和store进行连接。因为这个组件在Provider包裹下，而Provider下的所有组件都有能力去连接store
    • 让todolist组件和store做关联。store里的数据会映射到组件的props上面，同时，如果我想对store的数据做修改，我们可以通过store的props来进行修改

参数 mapStateToProps , mapDispatchToProps

  • mapStateToProps 此函数固定写法，会接收一个state（store里的数据） 然后return一个对象出去。
  • 放在connect第一个参数里面，意为 把store里的数据（state）映射给这个组件，变成组件的Props
    *
  • mapDispatchToProps Dispatch指发布，也就是store.dispatch. [store.dispatch挂载到Props上 ]
  • src/store/reducer.js
